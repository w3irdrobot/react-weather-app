What is React?

A JavaScript library for building user interfaces.

What does that mean?

It allows us to create interactive user interfaces in an easy to write and understand way.

React uses a virtual DOM to manage and batch changes to the actual DOM (more on this later).

We group pieces of our interface into components. These components can be composed of other components.

We then define how data is displayed based on the component's internal state and the props passed to it.

Let's define some terms.

Component

A piece of the UI managed by React. It has its own state that it manages internally and also has props that are passed into it.

This typically will directly represent markup in the browser.

State

Data saved directly on a component. No other component knows about this state, but this data can be passed to other components as props.

Props

Data passed to a component from its parent. We can use this data to respond to changes in state of the parent, among many other things.

Virtual DOM

An in-memory, JavaScript object representation of the DOM we see in the web browser.

When we change how a component looks through changes in data, we are actually changing what the calculated virtual DOM looks like. React will compare the new virtual DOM to the previous one it calculated and then update the DOM in the browser according to the differences in virtual DOM representations.

JSX

An HTML-like syntax for writing components.

React has a native API for creating components that is just calling a bunch of JavaScript functions. This works fine, but can become hairy when trying to maintain it.

With the power of Babel transpilation, we can write our components in JSX, which looks just like HTML. This makes reading and maintaining our components much easier.
